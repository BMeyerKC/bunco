Setting Up

1. Host Creates a Game

Everything that happens when the host sets up a new game.

flowchart TD A([Host opens game.html]) --> B[Fill out setup form\nHow many tables?\nHow many ghost slots?] B --> C[Click Create Game] C --> D[Generate a random 4-character code] D --> E[Save game to Firebase\nwith table count and host identity] E --> F[Add ghost players with random names\nto fill any empty seats] F --> G[Update URL with the new code] G --> H[Show waiting room\nwith shareable code] H --> I([Players can now join using the code])

2. Host Assigns Seats

How the host randomly places all players at tables before Round 1 starts.

flowchart TD A([Host is in waiting room]) --> B[Click Assign Seats Randomly] B --> C{Are all human seats filled?} C -- no --> D[Show warning: still waiting for players] D --> B C -- yes --> E[Shuffle all player IDs randomly\nincluding ghost players] E --> F[Assign each player a table, side, and seat\nevery 4 players fill one table] F --> G[Save assignments to Firebase\nunder round 1] G --> H[All players see their table number\nteammate and opponents in waiting room] H --> I([Host can now start Round 1])
Joining the Game

3. User Joins a Game

How a player gets from a share link into the waiting room.

flowchart TD A([Player receives share link with code]) --> B[Open the link] B --> C{Have you joined this game before?} C -- yes --> D[Player ID found on this device\nskip the form] C -- no --> E[Show the join form] E --> F[Player enters their name\nand clicks Join] F --> G[Check name is not already taken] G --> H[Add player to Firebase\nget back a player ID] H --> I[Save player ID to this device\nso you can return later] D --> J([Waiting room\nHost must start Round 1 to begin]) I --> J

4. Player Enters Name

What happens when a player submits the name form - including validation and error handling.

flowchart TD A([Player is on the join form]) --> B[Type a display name] B --> C[Click Join] C --> D[Load the game from Firebase] D --> E{Game found?} E -- no --> F[Show error: game not found\nCheck your code] F --> C E -- yes --> G{Is the name already taken\nby another player?} G -- yes --> H[Show error: name is taken\nTry adding an initial] H --> B G -- no --> I[Add player to Firebase\nreceive a unique player ID] I --> J[Save player ID to localStorage\nso this device is recognized] J --> K([Enter waiting room])

5. Late Join - Ghost Seat Claim

How a player joins a game that has already started by taking over a ghost seat.

flowchart TD A([Player opens join link while game is in progress]) --> B[Show in-progress options\nspectate link and ghost seat list] B --> C{Any ghost seats available?} C -- no --> D([Spectate only - go to live standings]) C -- yes --> E[Show list of open ghost seats\neach showing table number and teammate name] E --> F[Player clicks a seat to claim it] F --> G[Show confirmation form\nenter your name] G --> H[Click Take this seat] H --> I[Update the ghost player record in Firebase\nreplace ghost name with real name\nmark as not a ghost] I --> J[Save this player ID to the device\nusing the ghost ID as their own] J --> K([Go directly to the scoring view for that table])
Playing a Round

6. Host Starts a Round

What happens when the host clicks Start Round - triggering all players into the scoring view.

flowchart TD A([Host clicks Start Round 1]) --> B{Seats assigned for this round?} B -- no --> C[Show warning: assign seats first] B -- yes --> D[Create a score record in Firebase\nfor each table - scores start at 0] D --> E[Set currentRound to 1 in Firebase\nclear any previous game-called flag] E --> F[Firebase notifies all connected devices] F --> G[Each device sees currentRound is active\nand they are in the waiting room] G --> H[Every player navigates to the scoring view\nshowing their table and teammate names] H --> I([Round is live - players can start scoring])

7. Player Scores

What happens every time a player taps the scoring panel during a round.

flowchart TD A([Player is on the scoring view]) --> B[Tap the Us panel to add a point\nor tap the minus button to remove one] B --> C[Send increment or decrement to Firebase\nusing an atomic transaction so two taps never collide] C --> D[Firebase updates the live score] D --> E[watchTableScore fires on both devices at this table] E --> F[Both players see the updated score instantly] F --> G{Score reached 21 or more?} G -- yes --> H[Score panel turns bright blue or orange\nvisual indicator the round may be over] G -- no --> I[Continue tapping] H --> I I --> A

8. Player Submits Scores

How a table locks in their final score and signals they are done with the round.

flowchart TD A([Player decides the round is over at their table]) --> B[Optional: click Call Game\nlets other tables know to finish up] B --> C[Click Submit Scores] C --> D[Write final Us and Them scores to Firebase\nmark this table as submitted] D --> E[Show the submitted view\nwith a link to live standings] E --> F[Wait while other tables finish] F --> G{All tables submitted?} G -- no --> F G -- yes --> H([Host auto-advances to the next round])
Advancing and Ending

9. Round Auto-Advances

What the host device does automatically once every table has submitted their scores.

flowchart TD A([Host device gets a Firebase update]) --> B[Check if all tables have submitted\nfor the current round] B -- not all submitted --> C([Wait for the next update]) B -- all submitted --> D[Calculate who won at each table\nties go to the Us side] D --> E[Update each players cumulative record\nwins, losses, points, and buncos] E --> F[Save updated standings to Firebase] F --> G{Was that Round 6?} G -- no --> H[Calculate new seating\nwinners stay and split sides\nlosers rotate toward table 1\ntable 1 losers wrap to the last table] H --> I[Save new seat assignments to Firebase] I --> J[Advance currentRound by 1 in Firebase] J --> K([All players navigate to scoring for the new round]) G -- yes --> L([Game over - trigger final standings redirect])

10. Game Ends

What happens after all tables submit their Round 6 scores.

flowchart TD A([All tables submit Round 6 scores]) --> B[Host calculates final standings\nwins, losses, points, buncos for every player] B --> C[Save final standings to Firebase] C --> D[Set currentRound to 7 in Firebase\n7 is the game-complete signal] D --> E[Firebase notifies all connected devices] E --> F[Every device detects currentRound is 7] F --> G([All players redirect to standings.html\nwith the final flag set])
